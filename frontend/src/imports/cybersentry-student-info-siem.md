1. SIEM (Security Information and Event Management)
A SIEM is a system that collects, centralizes, and analyzes security- and system-related events generated by applications, servers, and user activities. In CyberSentry, the SIEM component focuses on event collection, correlation, visibility, and alerting rather than predictive analytics. It supports real-time awareness of access attempts, data modifications, and policy-relevant activities affecting student and academic records. The SIEM module enables administrators to detect unusual behavior, investigate incidents, and maintain accountability through centralized logging and monitoring.

🔎 How Attackers Test Vulnerabilities (Including OWASP ZAP & Burp Suite)
OWASP ZAP – Automated Web Application Scanning
Goal: Automatically detect common web vulnerabilities.
Step-by-step (conceptual):
The attacker enters the target website URL.


ZAP performs a crawl (spidering) of the website.


It identifies:


Login forms


Input fields


URL parameters


It automatically sends test payloads to:


Forms


Query parameters


It analyzes responses for:


SQL Injection signs


Cross-Site Scripting (XSS)


CSRF vulnerabilities


Security misconfigurations


What they’re thinking:
“Is this website vulnerable to common web attacks?”
 “Did the developer properly validate inputs?”
Burp Suite (Community Edition) – Manual Web Security Testing
Goal: Intercept and manipulate HTTP requests.
Step-by-step (conceptual):
The attacker configures their browser to pass traffic through Burp.


They log in or interact with the website.


Burp captures the HTTP request and response.


The attacker modifies request values:


Change user ID numbers


Modify parameters


Alter session tokens


They resend the modified request to the server.


They observe if:


Authorization checks are weak


Access control is broken


Validation is missing


What they’re thinking:
“Can I access another user’s data by changing this ID?”
 “Is the server validating this request properly?”
This is commonly used to test:
Broken Access Control


Insecure Direct Object Reference (IDOR)


Session management flaws


Nmap – Network Scanning Phase
Goal: Discover entry points into the system.
Step-by-step (conceptual):
The attacker identifies the target IP address or domain.


They scan the server to see which ports are open.


They check what services are running (e.g., web server, database).


They identify the software versions.


They look for:


Unnecessary open ports


Outdated software


Exposed database ports


What they’re thinking:
“Which doors are open?”
 “Is there a service I can attack?”
If they see an exposed database port → they may attempt database attacks next.

Nikto – Web Server Vulnerability Scanning
Goal: Find weak server configurations.
Step-by-step (conceptual):
The attacker targets the website URL.


The scanner checks for:


Outdated Apache/Nginx versions


Default admin pages


Backup files (e.g., .bak, .old)


Misconfigured directories


It compares findings against a vulnerability database.


It generates a report of weaknesses.


What they’re thinking:
“Is this server outdated?”
 “Did the developer forget to remove test files?”
If outdated software is detected → they search for known exploits.

SQLmap – SQL Injection Testing
Goal: Check if input fields are vulnerable to SQL injection.
Step-by-step (conceptual):
The attacker finds an input field:


Login form


Search bar


URL parameter


They test whether the input is properly validated.


The tool sends specially crafted test inputs.


If the system responds abnormally, it means:


Input validation is weak


Queries are not parameterized


The attacker may attempt to:


Bypass login


Extract database tables


Dump user credentials


What they’re thinking:
“Can I manipulate the database through this form?”
 “Is the system filtering user input properly?”
If input isn’t sanitized → database data may be exposed.

 Wireshark – Network Traffic Sniffing
Goal: Intercept sensitive data in transit.
Step-by-step (conceptual):
The attacker connects to the same network (e.g., public WiFi).


They monitor network traffic.


They observe packets being transmitted.


They check whether:


Data is encrypted (HTTPS)


Credentials appear readable


If traffic is unencrypted:


Login credentials may be captured


Session cookies may be stolen
